Class: Player

(abstract) Player()

Base class for all participants in an Ensemble.

A Player represents a live, asynchronous actor that can be started and stopped via play() and pause(). It participates in an Ensemble, emits property change events, and is designed for dynamic composition and coordination.

Subclasses must implement play() and pause(). The base implementation just set playing = true/false;

Constructor

(abstract) new Player()

  • Player#event:propertyChanged

let(name, value) → {this}

Updates a public property and emits propertyChanging and propertyChanged events.

This method is intended for use with public fields only. It compares the current value to the new value, and if different:

  • Emits a propertyChanging event with the old and new values.
  • Sets the new value.
  • Emits a propertyChanged event with the old and new values.

notify(event, …args)

Emits an event and logs it to the console in a structured, readable way.

  • Logs the event name, timestamp, and all arguments.
  • Objects are logged directly (not stringified), preserving their structure.
  • Errors are clearly shown in the log output.
Parameters:
Name Type Attributes Description
event string

The event name (e.g., "info", "error", "state").
args any <repeatable>

Arguments to log and emit with the event.
Example
this.notify("info", "Connected successfully");
this.notify("error", new Error("Connection failed"));
this.notify("state", { id: "sprite42", vibes: 5 });

play()

Simply sets this.playing = true when activated. Subclasses must implement their own logic.
